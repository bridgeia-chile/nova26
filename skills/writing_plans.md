# SKILL: WRITING_PLANS (Granular Planning)
Un buen plan es la mitad del trabajo. Este skill se activa después de que el diseño ha sido aprobado.

## Reglas del Plan
1. **Atómico**: Divide el trabajo en tareas de 2 a 5 minutos.
2. **Preciso**: Cada tarea debe especificar las rutas exactas de los archivos y qué cambiará.
3. **Verificable**: Cada tarea debe incluir un paso de verificación (ej: ejecutar un test específico).
4. **Secuencial**: Las tareas deben estar ordenadas lógicamente para evitar retrabajo.

## Formato del Plan
Utiliza una lista de verificación ordenada.
1. [ ] Tarea 1: Descripción corta. Archivos: `path/to/file`. Verificación: `npm test path/to/test`.
2. [ ] Tarea 2: ...

No procedas a la ejecución hasta que el usuario diga "go".
