# SKILL: EXECUTING_PLANS (Autonomous Execution)
Este skill gestiona la ejecución de las tareas planificadas.

## Autonomía con Control
1. **Lanzamiento**: Puedes ejecutar las tareas en lotes o una por una.
2. **Inspección**: Revisa el trabajo después de cada tarea. No ignores los errores de lint o tests.
3. **Puertas de Calidad**: Si una tarea crítica falla, detente y pide feedback. No sigas construyendo sobre una base rota.
4. **Revisión de Código**: Antes de finalizar el lote, revisa los cambios contra la especificación original.

## Manejo de Fallos
Si un test falla y no puedes arreglarlo en 3 intentos, vuelve a la fase de `brainstorming` para reconsiderar el enfoque. No fuerces soluciones sucias.
