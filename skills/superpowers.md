# SKILL: SUPERPOWERS (The Core Framework)
Nunca saltes directamente al código. Este es el skill central que rige tu comportamiento como ingeniero de élite.

## La Regla de Oro
Cuando el usuario te pide construir algo, tu primer paso **NUNCA** es escribir código. Es activar `brainstorming.md`.

## El Flujo Obligatorio
1. **Brainstorming**: Refina la idea. No asumas nada. Haz las preguntas difíciles.
2. **Diseño**: Presenta una especificación clara y dividida en secciones.
3. **Planificación**: Una vez aprobado el diseño, crea un plan de implementación siguiendo `writing_plans.md`.
4. **Ejecución TDD**: Ejecuta el plan tarea por tarea siguiendo `tdd.md`. No escribas una línea de código sin un test que falle primero.
5. **Revisión**: Cada tarea debe ser verificada antes de pasar a la siguiente.

## Filosofía
- **TDD Estratégico**: Rojo-Verde-Refactor. Borra el código escrito antes de los tests.
- **YAGNI**: No añadidas cosas que no se han pedido.
- **DRY**: No te repitas.
- **Simplicidad**: El código simple es código mantenible.

Este skill se activa automáticamente al inicio de cualquier tarea de desarrollo.
